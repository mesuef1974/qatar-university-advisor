الملخص التنفيذي

أُجري تقييم بصري عميق للتطبيق عبر فحص حي لجميع الشاشات (موافقة، جنسية، دردشة، جامعات، مقارنة، مفضلة، قائمة جانبية) + تحليل كود التصميم (theme.js, index.css, 12 مكون React) عبر 10 محاور تصميمية.

النتيجة: التطبيق يعمل بهوية قطرية أصيلة (كستنائي + ذهبي) مع بنية تصميم متينة (7.27/10)، لكنه يُعاني من 3 أخطاء فورية (theme-color أزرق، focus ring مكسور، فونت خاطئ في الموافقة) وغياب Dark Mode وتجربة Desktop ضائعة.

S

نقاط قوة التصميم (9)

هوية

هوية قطرية أصيلة وموثقة

theme.js يصف #8A1538 صراحةً كـ "Qatar official maroon". ثنائية الكستنائي-الذهبي ليست عشوائية بل نظام مقرر.

PDPPL

شاشة الموافقة — من أفضل تصاميم PDPPL في المنطقة

Glass-morphism + backdrop-filter:blur(10px) + 4 نقاط خصوصية مصوَّرة + زر رفض ظاهر. تصميم يحترم المستخدم والقانون.

Animation

نظام Animation مدروس

5 Keyframes موثقة (bounce, msgIn, menuPulse, fadeIn, slideUp) + duration tokens + easing tokens في theme.js.

RTL

دعم RTL من الجذر

html[dir="rtl"] + body direction:rtl + كل المكونات تحترم الاتجاه. فقاعات الدردشة تتبدل تلقائياً.

UX

نبضة menuPulse — Interaction Design ذكي

نقطة ذهبية نابضة فوق الهامبرغر توجّه العين بدون نص. تصميم غير لفظي متقن.

Input

Input Bar — 3 مستويات Feedback

تحول لوني عند التركيز + زر يتحول من رمادي لكستنائي + spring animation. ثلاث إشارات في عنصر واحد.

Cards

بطاقات الجامعات متماسكة

حد جانبي ملوّن حسب النوع + أيقونة في دائرة + بادجات + expand/collapse. تدرج معلوماتي صحيح.

PWA

دعم PWA و iOS safe areas

viewport-fit=cover + env(safe-area-inset-bottom) + apple-mobile-web-app-capable. جاهز للنشر كـ PWA.

a11y

reduced-motion صحيح

@media (prefers-reduced-motion:reduce) يُلغي كل animations — حماية شاملة لذوي الحساسية للحركة.

W

نقاط ضعف التصميم (9)

حرج

theme-color أزرق #1a56db في index.html

سطران meta theme-color — الثاني أزرق من قالب Vite لم يُنظَّف. شريط iOS يظهر بأزرق بدل الكستنائي. يحتاج ثانية لإصلاحه.

حرج

PrivacyConsent يستخدم Segoe UI بدل Tajawal

أول شاشة يراها المستخدم تعمل بفونت ويندوز. الانطباع الأول مكسور هوياتياً.

حرج

Focus ring أزرق #1a56db يكسر نظام الألوان

accessibility.css يُعيد تعريف outline بأزرق. أول مستخدم لوحة مفاتيح يرى لوناً خارج النظام.

عالي

لا Dark Mode في 2026

color-scheme:light صريحة بلا بديل. 81% من مستخدمي الجوال يفضلون الوضع الداكن.

عالي

تجربة Desktop ضائعة

على 1280px+ التطبيق = هاتف مكبَّر وسط خلفية فارغة. لا two-column layout. فرصة حاسب جامعي ضائعة.

عالي

theme.js توكن ميت جزئياً

ألوان مُعرَّفة في theme لكن المكونات تكتب القيم hardcoded. النظام موجود لكن غير مُتبَّع (~40%).

متوسط

نصوص 9-10px — فشل WCAG

بادج المفضلة 9px، timestamps 10px. غير مقروءة بدون تكبير. فشل WCAG 1.4.4.

متوسط

SideMenu قائمة طويلة بلا تجميع

23+ جامعة كقائمة نصية مسطحة. بدون search أو virtual scroll أو تصنيف.

متوسط

لا Skeleton Loading

isTyping indicator جيد، لكن لا skeleton للتحميل الأولي. فراغ أو flash عند الفتح.

O

فرص التصميم (7)

سريعة

Dark Mode بـ CSS Variables — ساعة عمل

المتغيرات في :root جاهزة. @media (prefers-color-scheme:dark) بإعادة تعريفها = تحسين فوري. --bg:#1C1414, --surface:#2D1B1B.

Desktop

Two-Column Layout لـ 1024px+

يسار = جامعات مع فلاتر، يمين = محادثة. نمط ChatGPT. البنية جاهزة — يحتاج layout wrapper فقط.

Brand

Logo SVG بدلاً من Emoji

استبدال emoji الخريج بلوجو SVG مخصص (قبة + نجمة قطرية). يرفع الاحترافية ويُستخدم كـ favicon + PWA icon.

Chat

Markdown Rendering محسّن

إضافة جداول + قوائم مرقمة + headers + links قابلة للنقر. يجعل الردود أقرب لـ ChatGPT/Perplexity.

Mobile

Haptic Feedback مجاني

navigator.vibrate(8) عند الإرسال أو المفضلة. تجربة أقرب للـ Native بدون تكلفة.

UX

Scroll Indicator للفلاتر

gradient fade على الجانبين يُشير لمحتوى مخفي. نمط Netflix/YouTube.

Empty

SVG Illustrations بدل Emoji في Empty States

رسومات حقيقية بدل emoji 48px. هوية بصرية أقوى في صفحات المقارنة والمفضلة الفارغة.

T

تهديدات التصميم (5)

قانوني

WCAG 2.2 — خطر في audit حكومي

9px text + focus مكسور + emoji بلا aria = فشل محتمل. جهات حكومية قطرية قد تطلب WCAG AA.

تجربة

Android Dark Mode — شاشة بيضاء ليلاً

color-scheme:light المُجبَرة = تطبيق ساطع على جهاز مظلم. تجربة مؤلمة تدفع للإغلاق.

منافسة

ChatGPT بالعربية — markdown كامل + جداول

بدون قيمة بصرية وتفاعلية واضحة (مقارنة بصرية، خرائط) سيصعب التمييز.

دين تقني

Inline Styles = تغيير شامل مستحيل

30-40% من theme.js مُتبَّع فقط. الباقي hardcoded. أي تغيير لوني = تعديل عشرات الملفات.

هوية

FontFamily غير موحدة عبر المكونات

3 أنماط: Segoe UI (Consent) + Tajawal (معظم) + Cairo (عناوين). مستخدم iOS يرى San Francisco في بعض الشاشات.

التوصيات — مرتبة حسب الأولوية والتأثير

فوري — 5 دقائق

إصلاح theme-color + focus ring + font

3 أسطر تُصلح 3 أخطاء: حذف meta أزرق، تغيير outline لكستنائي، إضافة Tajawal لـ PrivacyConsent.

عالي — ساعة

إضافة Dark Mode عبر CSS Variables

@media (prefers-color-scheme:dark) { :root { --bg:#1C1414; --surface:#2D1B1B; --text:#F5F0EB; } } — يرفع التقييم +1.5 نقطة.

عالي — يوم

Desktop Two-Column Layout

على 1024px+: يسار=جامعات، يمين=محادثة. يفتح شريحة مستخدمي الحاسب الجامعي.

متوسط

رفع أحجام النصوص الصغيرة لـ 12px+

9-10px → 12px minimum. يُصلح WCAG 1.4.4 ويُحسن القراءة.

متوسط

توحيد theme.js token adoption

استبدال كل hardcoded colors/spacing في المكونات بمراجع theme. الهدف: 90%+ adoption.

متوسط

Logo SVG + Markdown Rendering

لوجو مخصص + جداول/قوائم في الردود = قفزة احترافية.
