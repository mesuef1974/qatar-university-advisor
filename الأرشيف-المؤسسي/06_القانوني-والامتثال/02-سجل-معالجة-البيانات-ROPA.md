## مقدمة

يُعدّ هذا السجل وثيقة إلزامية بموجب المادة 19 من قانون حماية البيانات الشخصية القطري (القانون رقم 13 لسنة 2016). يوثّق جميع أنشطة معالجة البيانات الشخصية التي تجريها شركة النخبوية للبرمجيات في إطار مشروع **مستشار جامعة قطر**، ويُشكّل دليلاً على الامتثال للجهات الرقابية عند الطلب.

**معلومات المتحكم في البيانات (Data Controller):**
- **الاسم:** شركة النخبوية للبرمجيات
- **العنوان:** الدوحة، دولة قطر
- **مسؤول حماية البيانات:** [dpo@elite-software.qa]

## النشاط 01: تسجيل المستخدم (User Registration)

| الحقل | التفاصيل |
|---|---|
| **اسم النشاط** | تسجيل المستخدم وإنشاء الحساب |
| **الغرض** | إنشاء هوية فريدة للمستخدم وتمكينه من الوصول إلى خدمة الإرشاد الأكاديمي |
| **فئات أصحاب البيانات** | طلاب جامعيون في قطر (18+ سنة) |
| **فئات البيانات الشخصية** | رقم هاتف واتساب، UUID داخلي، اسم المستخدم (اختياري)، تاريخ ووقت التسجيل، بيانات إعدادات اللغة |
| **المستلمون / المعالجون** | Supabase (تخزين)، Meta WhatsApp (استقبال الرقم)، Vercel (استضافة) |
| **فترة الاحتفاظ** | سنتان (24 شهراً) من تاريخ آخر نشاط، ثم إخفاء هوية |
| **الأساس القانوني** | موافقة صريحة — PDPPL المادة 7(أ) |
| **النقل لدول أخرى** | نعم |
| **دول النقل** | الولايات المتحدة (Supabase AWS us-east-1، Vercel) |
| **ضمانات النقل** | اتفاقيات DPA + تشفير AES-256 + TLS 1.3 |
| **تقييم مخاطر DPIA** | غير مطلوب (خطورة منخفضة) |

## النشاط 02: تسجيل المحادثات (Conversation Logging)

| الحقل | التفاصيل |
|---|---|
| **اسم النشاط** | تسجيل وحفظ سجل المحادثات |
| **الغرض** | حفظ سياق المحادثة لتقديم إرشاد أكاديمي متواصل ومتسق، وتمكين المستخدم من استعراض تاريخ محادثاته |
| **فئات أصحاب البيانات** | طلاب جامعيون مسجلون في النظام |
| **فئات البيانات الشخصية** | نص الرسائل (المستخدم + النظام)، معرّف الجلسة، طوابع الوقت، رقم هاتف المستخدم (مربوط بالمحادثة)، معرّف المحادثة |
| **المستلمون / المعالجون** | Supabase (تخزين)، Google Gemini API (معالجة النص لتوليد الرد)، Vercel (معالجة الطلبات) |
| **فترة الاحتفاظ** | سنتان (24 شهراً) من تاريخ كل محادثة، ثم حذف نهائي |
| **الأساس القانوني** | موافقة صريحة — PDPPL المادة 7(أ) + تنفيذ الخدمة المتفق عليها — المادة 7(ب) |
| **النقل لدول أخرى** | نعم |
| **دول النقل** | الولايات المتحدة (Google، Supabase، Vercel) |
| **ضمانات النقل** | Google Cloud DPA + Supabase DPA + Vercel DPA + تشفير كامل |
| **تقييم مخاطر DPIA** | **مطلوب** — يُجرى بسبب معالجة محتوى المحادثات عبر نماذج AI |
| **ملاحظة** | لا تستخدم Google بيانات المحادثات لتدريب النماذج وفق اتفاقية Google Cloud DPA |

## النشاط 03: الإرشاد الأكاديمي (Academic Advising)

| الحقل | التفاصيل |
|---|---|
| **اسم النشاط** | تقديم الإرشاد الأكاديمي المخصص |
| **الغرض** | تحليل الوضع الأكاديمي للطالب وتقديم توصيات مخصصة بشأن المقررات وخطة الدراسة والمتطلبات |
| **فئات أصحاب البيانات** | طلاب جامعيون مسجلون |
| **فئات البيانات الشخصية** | التخصص الدراسي، المقررات المكتملة/المسجلة، المعدل التراكمي (GPA)، الساعات المعتمدة، اسم الجامعة، المستوى الدراسي |
| **المستلمون / المعالجون** | Supabase (قراءة البيانات)، Google Gemini API (التحليل والتوصيات)، Vercel (تنفيذ المنطق) |
| **فترة الاحتفاظ** | سنتان من تاريخ آخر تحديث للبيانات الأكاديمية، ثم إخفاء هوية |
| **الأساس القانوني** | موافقة صريحة — PDPPL المادة 7(أ) |
| **النقل لدول أخرى** | نعم |
| **دول النقل** | الولايات المتحدة |
| **ضمانات النقل** | DPA مع جميع المعالجين + بيانات تُرسل للـ AI مجردة من معرفات التعريف قدر الإمكان |
| **تقييم مخاطر DPIA** | **مطلوب** — معالجة AI لبيانات أكاديمية حساسة قد تؤثر على قرارات الطالب |

## النشاط 04: التحليلات (Analytics)

| الحقل | التفاصيل |
|---|---|
| **اسم النشاط** | تحليل استخدام النظام والتحسين المستمر |
| **الغرض** | قياس أداء النظام، فهم أنماط الاستخدام، تحديد مجالات التحسين |
| **فئات أصحاب البيانات** | جميع مستخدمي النظام (مجهولو الهوية بعد المعالجة) |
| **فئات البيانات الشخصية** | عدد الجلسات، أنواع الاستفسارات (مُصنَّفة)، أوقات الاستخدام، معدلات الاستجابة — **لا تُستخدم بيانات تعريفية مباشرة** |
| **المستلمون / المعالجون** | Supabase (استعلامات إحصائية)، Vercel Analytics (مقاييس الأداء) |
| **فترة الاحتفاظ** | سنة واحدة للبيانات المجمّعة، ثم حذف نهائي |
| **الأساس القانوني** | مصلحة مشروعة — PDPPL المادة 7(د) (تحسين الخدمة) + موافقة المستخدم على شروط الاستخدام |
| **النقل لدول أخرى** | نعم (بيانات مجمّعة مجهولة الهوية فقط) |
| **دول النقل** | الولايات المتحدة (Vercel) |
| **ضمانات النقل** | Vercel DPA + إخفاء الهوية الكامل قبل المعالجة |
| **تقييم مخاطر DPIA** | غير مطلوب (بيانات مجهولة الهوية) |

## النشاط 05: وظيفة CRON لتنظيف البيانات (PDPPL Cleanup Cron)

| الحقل | التفاصيل |
|---|---|
| **اسم النشاط** | الحذف التلقائي للبيانات المنتهية فترة احتفاظها |
| **الغرض** | الامتثال لمتطلبات PDPPL بشأن عدم الاحتفاظ بالبيانات أطول من الضرورة؛ تنفيذ مبدأ تقليل البيانات (Data Minimisation) |
| **فئات أصحاب البيانات** | جميع المستخدمين الذين تجاوزت بياناتهم فترة الاحتفاظ المحددة |
| **فئات البيانات الشخصية** | جميع فئات البيانات المُحتفظ بها (للتحقق من المدة وتنفيذ الإجراء) |
| **المستلمون / المعالجون** | Supabase (تنفيذ الحذف/إخفاء الهوية) |
| **فترة الاحتفاظ** | سجلات تنفيذ الوظيفة: سنة واحدة |
| **الأساس القانوني** | التزام قانوني — PDPPL المادة 9 (مبدأ تحديد الغرض والتقليل من البيانات) |
| **النقل لدول أخرى** | لا (عملية داخلية ضمن Supabase) |
| **ضمانات النقل** | غير مطبقة |
| **تقييم مخاطر DPIA** | غير مطلوب |
| **جدول التشغيل** | اليوم الأول من كل شهر ميلادي (Cron: `0 2 1 * *` بتوقيت UTC) |

## النشاط 06: مراسلة واتساب (WhatsApp Messaging)

| الحقل | التفاصيل |
|---|---|
| **اسم النشاط** | استقبال وإرسال الرسائل عبر WhatsApp Business API |
| **الغرض** | توفير قناة التواصل الرئيسية مع المستخدمين لتقديم الإرشاد الأكاديمي |
| **فئات أصحاب البيانات** | طلاب جامعيون يستخدمون واتساب في قطر |
| **فئات البيانات الشخصية** | رقم هاتف واتساب، نص الرسائل الواردة والصادرة، معرفات الرسائل (Message IDs)، طوابع الوقت، حالة التسليم (Delivered/Read) |
| **المستلمون / المعالجون** | Meta (WhatsApp Business Platform)، Vercel (Webhook استقبال الرسائل)، Supabase (تخزين الرسائل)، Google Gemini (معالجة المحتوى) |
| **فترة الاحتفاظ** | سنتان لسجلات الرسائل، ثم حذف نهائي |
| **الأساس القانوني** | موافقة صريحة — PDPPL المادة 7(أ) (الموافقة على استخدام واتساب كقناة تواصل) |
| **النقل لدول أخرى** | نعم |
| **دول النقل** | الولايات المتحدة وأوروبا (بنية تحتية Meta العالمية) |
| **ضمانات النقل** | شروط Meta Business Platform + تشفير end-to-end للرسائل + Meta DPA |
| **تقييم مخاطر DPIA** | **مطلوب** — بسبب طبيعة بيانات الرسائل الشخصية ومشاركتها مع Meta |
