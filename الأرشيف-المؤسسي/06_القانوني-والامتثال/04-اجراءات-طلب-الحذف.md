## 1. الغرض من هذا الإجراء

يُحدد هذا الإجراء التشغيلي الخطوات الواجب اتباعها عند استلام طلب حذف بيانات شخصية من مستخدم لمشروع **مستشار جامعة قطر**. يضمن هذا الإجراء:
- الامتثال للمادة 16 من PDPPL (الحق في النسيان).
- التعامل المنتظم والموثق مع كل طلب.
- اكتمال عملية الحذف أو إخفاء الهوية خلال الإطار الزمني المحدد قانونياً.
- الحفاظ على سجل تدقيق كامل دون الاحتفاظ بالبيانات المحذوفة.

---

## 2. قنوات استقبال طلبات الحذف

### 2.1 عبر واتساب (القناة الرئيسية)
يستطيع المستخدم إرسال أي من الأوامر التالية:

```
احذف بياناتي
حذف حسابي
delete my data
أريد حذف حسابي
نسيان
```

يقوم النظام تلقائياً بـ:
1. إرسال رسالة تأكيد استلام الطلب فوراً.
2. تسجيل الطلب في قاعدة بيانات طلبات الحذف.
3. إخطار DPO عبر البريد الإلكتروني.

### 2.2 عبر تطبيق الويب
- الانتقال إلى: **الإعدادات ← الخصوصية ← طلب حذف الحساب**
- ملء النموذج المخصص بما يتضمن: رقم الهاتف، سبب الطلب (اختياري)، بريد إلكتروني للتأكيد (اختياري).

### 2.3 عبر البريد الإلكتروني للـ DPO
- الإرسال إلى: **dpo@elite-software.qa** *(placeholder)*
- يجب أن يتضمن: اسم المستخدم أو رقم الهاتف المرتبط بالحساب.

## 3. الخط الزمني الكامل لعملية الحذف

```
الطلب واردٌ
     │
     ▼ (خلال دقائق — آلي)
[ساعة 0] استلام الطلب + رسالة تأكيد فورية للمستخدم
     │
     ▼ (خلال 24 ساعة)
[يوم 1] إخطار رسمي بالاستلام + تحديد هوية صاحب الطلب + توثيق في السجل
     │
     ▼ (مرحلة فترة الرجوع)
[أيام 2-7] فترة السماح (Grace Period) — 7 أيام يمكن فيها إلغاء الطلب
     │
     ▼ (اليوم 7 — إذا لم يُلغَ الطلب)
[يوم 7] بدء تنفيذ الحذف الفعلي
     │
     ├──► حذف البيانات القابلة للحذف الكامل
     ├──► إخفاء هوية البيانات التي تستلزم الاحتفاظ
     └──► حذف من المعالجين الخارجيين (Supabase، النسخ الاحتياطية)
     │
     ▼ (خلال ساعات من التنفيذ)
[يوم 7-8] إرسال رسالة تأكيد إتمام الحذف للمستخدم
     │
     ▼
[يوم 8] توثيق الحذف في سجل التدقيق وإغلاق الطلب
```

---

## 4. فترة السماح (Grace Period) — 7 أيام

**سبب فترة السماح:** منح المستخدم فرصة لإعادة النظر في قراره، لا سيما أن الحذف لا رجعة فيه.

**خلال فترة السماح:**
- يستمر الحساب في العمل بشكل طبيعي.
- يمكن للمستخدم إلغاء الطلب بإرسال: `"ألغِ طلب الحذف"` أو `"cancel delete request"`.
- تُرسل رسالة تذكير للمستخدم في اليوم الخامس.

**استثناءات فترة السماح (تنفيذ فوري):**
- طلبات الحذف الواردة من جهة قانونية مختصة.
- اكتشاف أن المستخدم قاصر (دون 18 سنة).
- طلب مرتبط بحادثة خصوصية مُبلَّغ عنها.

## 5. ما يتم حذفه مقابل ما يُخفى هويته

### 5.1 البيانات التي تُحذف حذفاً نهائياً وكاملاً

| البيانات | مبرر الحذف الكامل |
|---|---|
| رقم الهاتف (بعد إخفاء الهوية) | بيانات تعريفية مباشرة — لا سبب للاحتفاظ |
| نصوص جميع المحادثات | محتوى شخصي لا قيمة إحصائية فيه |
| البيانات الأكاديمية التفصيلية | بيانات خاصة جداً |
| بيانات الإعدادات والتفضيلات | لا حاجة بعد الحذف |
| رمز التحقق والمصادقة | بيانات أمان لا لزوم لها بعد الحذف |

**الأسلوب التقني:** `DELETE FROM` في قاعدة بيانات Supabase + overwrite آمن للبيانات المشفرة.

### 5.2 البيانات التي تُخفى هويتها (لا تُحذف)

| البيانات | سبب الاحتفاظ المشروع |
|---|---|
| سجل الموافقة (بدون معرّفات) | إثبات الامتثال القانوني — 5 سنوات |
| سجل طلب الحذف ذاته | إثبات تنفيذ حق PDPPL — 5 سنوات |
| إحصائيات الاستخدام المجمّعة | تحسين الخدمة — لا تتضمن بيانات شخصية |
| سجلات الأمان التقنية (مجهولة) | الأمن السيبراني — سنة واحدة |

**أسلوب إخفاء الهوية:** استبدال جميع المعرفات الشخصية بقيم عشوائية لا يمكن ربطها بالمستخدم الأصلي.

### 5.3 البيانات لدى المعالجين الخارجيين

| المعالج | الإجراء | الإطار الزمني |
|---|---|---|
| Supabase | حذف مباشر عبر قاعدة البيانات | فوري عند تنفيذ الحذف |
| النسخ الاحتياطية (Supabase Backups) | تُحذف مع دورة النسخ الاحتياطي التالية | خلال 90 يوماً |
| Vercel (سجلات Edge) | انتهاء صلاحية تلقائي | خلال 30 يوماً |
| Google Gemini (API) | لا يُخزَّن بشكل دائم وفق DPA | لا يُطلب إجراء |
| Meta WhatsApp | تطبيق المادة 12 من Meta Business Terms | خلال 30-90 يوماً |

## 6. قالب رسائل التواصل مع المستخدم

### 6.1 رسالة تأكيد استلام الطلب (فورية — آلية)

```
مرحباً،

استلمنا طلبكم بحذف بياناتكم الشخصية من مستشار جامعة قطر.

رقم الطلب: [DELETION-XXXX-XXXXXXXX]
تاريخ الاستلام: [التاريخ والوقت]

ستبدأ عملية الحذف خلال 7 أيام ما لم تطلبوا الإلغاء.
لإلغاء الطلب، أرسلوا: "ألغِ طلب الحذف"

نحترم خصوصيتكم وحقوقكم.
فريق مستشار جامعة قطر — شركة النخبوية للبرمجيات
```

### 6.2 رسالة التذكير قبل التنفيذ (اليوم الخامس)

```
تذكير مهم:

سيتم حذف بياناتكم من مستشار جامعة قطر خلال يومين (يوم [التاريخ]).

هذا الإجراء لا رجعة فيه. إذا غيّرتم رأيكم، أرسلوا:
"ألغِ طلب الحذف" قبل [التاريخ والوقت].

رقم الطلب: [DELETION-XXXX-XXXXXXXX]
```

### 6.3 رسالة تأكيد إتمام الحذف

```
تأكيد الحذف الكامل

تم حذف جميع بياناتكم الشخصية من مستشار جامعة قطر بنجاح.

ما تم حذفه:
✓ رقم هاتفكم وبيانات التعريف
✓ جميع سجلات المحادثات
✓ البيانات الأكاديمية

ملاحظة: قد يستغرق الحذف من النسخ الاحتياطية حتى 90 يوماً.

شكراً لاستخدامكم خدمتنا.
فريق مستشار جامعة قطر — شركة النخبوية للبرمجيات
```

## 7. التوثيق في سجل التدقيق

لكل طلب حذف يُسجَّل ما يلي في جدول `deletion_audit_log`:

| الحقل | القيمة المُسجَّلة |
|---|---|
| `request_id` | معرّف فريد للطلب (DELETION-YYYY-XXXXXXXX) |
| `request_channel` | واتساب / ويب / بريد إلكتروني |
| `request_timestamp` | تاريخ ووقت الطلب (UTC) |
| `user_id_hash` | هاش غير قابل للعكس لمعرّف المستخدم |
| `acknowledgment_timestamp` | تاريخ الإقرار الرسمي |
| `execution_timestamp` | تاريخ تنفيذ الحذف الفعلي |
| `deleted_records_count` | عدد السجلات المحذوفة |
| `anonymized_records_count` | عدد السجلات التي أُخفيت هويتها |
| `external_processors_notified` | قائمة المعالجين الخارجيين الذين تم إخطارهم |
| `confirmation_sent` | هل أُرسل تأكيد للمستخدم؟ (نعم/لا) |
| `handled_by` | معرّف الموظف المسؤول (إن لم يكن آلياً) |

## 8. حالات الرفض المشروعة لطلب الحذف

وفق المادة 16 من PDPPL، يمكن رفض طلب الحذف في الحالات التالية:

| سبب الرفض | المرجع القانوني | الإجراء |
|---|---|---|
| الالتزام بالتزام قانوني مستمر | المادة 16(أ) | إخطار المستخدم بالسبب وتحديد موعد الحذف |
| وجود نزاع قانوني قائم | المادة 16(ب) | إخطار المستشار القانوني + إخطار المستخدم |
| الحفاظ على أدلة لحماية حقوق قانونية | المادة 16(ج) | توثيق القرار مع مبرراته |

**في جميع حالات الرفض:** يجب إخطار المستخدم كتابةً بالسبب خلال 14 يوم عمل من استلام الطلب.
