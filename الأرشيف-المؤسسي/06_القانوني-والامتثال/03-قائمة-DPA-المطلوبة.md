## مقدمة

تُلزم المادة 12 من قانون PDPPL (القانون رقم 13 لسنة 2016) المتحكّم في البيانات (**شركة النخبوية للبرمجيات**) بإبرام اتفاقيات مكتوبة مع جميع معالجي البيانات الخارجيين. هذه الاتفاقيات يجب أن تنص على:

1. طبيعة ونطاق ومدة المعالجة.
2. التزام المعالج بتنفيذ تدابير الأمان المناسبة.
3. عدم استخدام البيانات لأغراض خارج نطاق التعاقد.
4. حق المتحكم في التدقيق وفسخ الاتفاقية.
5. التزام المعالج بدعم ممارسة حقوق أصحاب البيانات.

## المعالج 01: Supabase (قاعدة البيانات)

### معلومات المعالج

| الحقل | التفاصيل |
|---|---|
| **اسم الشركة** | Supabase, Inc. |
| **النوع** | مزود قاعدة بيانات PostgreSQL مُدارة (Database-as-a-Service) |
| **موقع الخوادم** | AWS us-east-1 (افتراضي)، مع خيارات أوروبية وآسيوية |
| **الحالة** | ⏳ قيد الانتظار (Pending) |
| **الموعد النهائي** | 2026-05-01 |

### البيانات المُعالَجة

- أرقام هواتف المستخدمين (مشفرة)
- سجلات المحادثات كاملة
- البيانات الأكاديمية للطلاب
- سجلات الموافقات
- سجلات التدقيق والأمان
- بيانات إعدادات المستخدمين

### خطوات توقيع اتفاقية DPA

1. **تسجيل الدخول** إلى لوحة تحكم Supabase: [app.supabase.com](https://app.supabase.com)
2. الانتقال إلى: **Organization Settings → Legal → Data Processing Agreement**
3. مراجعة نص الاتفاقية المُقدَّمة من Supabase.
4. إدخال بيانات الشركة:
   - اسم الشركة: شركة النخبوية للبرمجيات / Elite Software Company
   - العنوان: الدوحة، قطر
   - البريد الإلكتروني القانوني: legal@elite-software.qa
5. النقر على **"Sign DPA"** والتأكيد.
6. تنزيل نسخة PDF الموقعة وحفظها في هذا المجلد.
7. **بديل:** إرسال طلب عبر: [privacy@supabase.io](mailto:privacy@supabase.io) مع ذكر اسم المؤسسة ورقم العقد.

### الوثائق المرفقة المطلوبة

- [ ] نسخة PDF من DPA الموقعة
- [ ] شهادات الامتثال الحالية (SOC 2 Type II، ISO 27001)
- [ ] سياسة خصوصية Supabase السارية

## المعالج 02: Vercel (الاستضافة والخوادم اللاإدارية)

### معلومات المعالج

| الحقل | التفاصيل |
|---|---|
| **اسم الشركة** | Vercel, Inc. |
| **النوع** | منصة استضافة وخوادم لاإدارية (Serverless/Edge) |
| **موقع الخوادم** | شبكة Edge عالمية (US، أوروبا، آسيا) |
| **الحالة** | ⏳ قيد الانتظار (Pending) |
| **الموعد النهائي** | 2026-05-01 |

### البيانات المُعالَجة

- طلبات HTTP بما تتضمنه من عناوين IP وبيانات الطلب
- سجلات تشغيل Serverless Functions (قد تتضمن بيانات مستخدم)
- بيانات Edge Middleware (التحقق من المصادقة)
- مقاييس الأداء (Vercel Analytics)

### خطوات توقيع اتفاقية DPA

1. **تسجيل الدخول** إلى لوحة تحكم Vercel: [vercel.com/dashboard](https://vercel.com/dashboard)
2. الانتقال إلى: **Team Settings → Security & Privacy → Data Processing Agreement**
3. *(في الخطط المدفوعة Pro/Enterprise فقط)* مراجعة الاتفاقية المُقدَّمة.
4. **ملاحظة مهمة:** الخطة المجانية لا تتضمن DPA رسمية — يُنصح بالترقية إلى **Vercel Pro** قبل الإطلاق الإنتاجي.
5. إدخال بيانات الشركة وتوقيع الاتفاقية إلكترونياً.
6. **بديل للاتصال المباشر:** [dpa@vercel.com](mailto:dpa@vercel.com)
7. يمكن الاطلاع على نموذج DPA المعياري في: [vercel.com/legal/dpa](https://vercel.com/legal/dpa)

### الوثائق المرفقة المطلوبة

- [ ] نسخة PDF من DPA الموقعة
- [ ] تأكيد اشتراك Vercel Pro أو Enterprise
- [ ] قائمة Subprocessors الخاصة بـ Vercel

## المعالج 03: Google Cloud / Gemini API

### معلومات المعالج

| الحقل | التفاصيل |
|---|---|
| **اسم الشركة** | Google LLC / Alphabet Inc. |
| **الخدمة المستخدمة** | Google Gemini API (عبر Google AI Studio أو Vertex AI) |
| **موقع الخوادم** | الولايات المتحدة (مراكز Google العالمية) |
| **الحالة** | ⏳ قيد الانتظار (Pending) |
| **الموعد النهائي** | 2026-05-01 |

### البيانات المُعالَجة

- نصوص محادثات المستخدمين (تُرسل للنموذج لتوليد الردود)
- البيانات الأكاديمية المضمّنة في Prompts
- لا يتم تخزين البيانات بشكل دائم لدى Google وفق اتفاقية Google Cloud DPA

### خطوات توقيع اتفاقية DPA

#### الخيار أ: Google AI Studio (المسار الحالي)
1. **تسجيل الدخول** إلى: [aistudio.google.com](https://aistudio.google.com)
2. ملاحظة: Google AI Studio **لا توفر DPA رسمية** في الخطط المجانية.
3. **التوصية الحتمية:** الترقية إلى **Google Cloud Vertex AI** للحصول على DPA رسمية.

#### الخيار ب: Google Cloud / Vertex AI (المسار الموصى به للإنتاج)
1. إنشاء حساب Google Cloud: [console.cloud.google.com](https://console.cloud.google.com)
2. الانتقال إلى: **IAM & Admin → Privacy → Data Processing Terms**
3. قبول **Google Cloud Data Processing Addendum (CDPA)** عبر لوحة التحكم.
4. أو تحميل DPA من: [cloud.google.com/terms/data-processing-addendum](https://cloud.google.com/terms/data-processing-addendum)
5. توقيع الوثيقة وإرسالها إلى Google عبر: [privacy-policy-questions@google.com](mailto:privacy-policy-questions@google.com)

#### معلومات مهمة بشأن Gemini API
- يجب تفعيل **"Do not use my data for model training"** في إعدادات Google AI Studio.
- في Vertex AI، هذا الإعداد مفعّل افتراضياً بموجب شروط المؤسسات.

### الوثائق المرفقة المطلوبة

- [ ] قبول Google Cloud Data Processing Addendum
- [ ] لقطة شاشة لإعداد "No training on data"
- [ ] نسخة من شروط Gemini API السارية

## المعالج 04: Meta / WhatsApp Business API

### معلومات المعالج

| الحقل | التفاصيل |
|---|---|
| **اسم الشركة** | Meta Platforms, Inc. |
| **الخدمة المستخدمة** | WhatsApp Business API (Cloud API) |
| **موقع الخوادم** | بنية تحتية Meta العالمية (US + أوروبا) |
| **الحالة** | ⏳ قيد الانتظار (Pending) |
| **الموعد النهائي** | 2026-05-01 |

### البيانات المُعالَجة

- أرقام هواتف المستخدمين
- محتوى الرسائل النصية (مشفرة end-to-end من جهة المستخدم)
- البيانات الوصفية للرسائل (وقت الإرسال، حالة التسليم)
- بيانات حساب WhatsApp Business

### خطوات توقيع اتفاقية DPA

1. **تسجيل الدخول** إلى Meta Business Manager: [business.facebook.com](https://business.facebook.com)
2. الانتقال إلى: **Business Settings → Data Sources → WhatsApp Accounts**
3. مراجعة وقبول **Meta Business Tools Terms** (تتضمن أحكام معالجة البيانات).
4. **اتفاقية DPA الرسمية:** متاحة ضمن **WhatsApp Business Solution Provider Agreement** للشركاء المعتمدين.
5. الاطلاع على اتفاقية معالجة البيانات الكاملة في:
   - [www.whatsapp.com/legal/business-data-processing-terms](https://www.whatsapp.com/legal/business-data-processing-terms)
6. للحصول على اتفاقية مخصصة (Enterprise):
   - التواصل مع Meta Business Sales: [business.facebook.com/contact](https://business.facebook.com/contact)

#### ملاحظة بشأن Cloud API مقابل On-Premise
- **Cloud API (المستخدم حالياً):** Meta تعمل كمعالج جزئي للبيانات — شروط معالجة البيانات محددة في **WhatsApp Business Terms**.
- **On-Premise API:** الشركة تتحكم في الخوادم — مسؤولية أعلى لكن تحكم أكبر.

### الوثائق المرفقة المطلوبة

- [ ] قبول Meta Business Tools Terms (لقطة شاشة + تاريخ)
- [ ] نسخة من WhatsApp Business Data Processing Terms
- [ ] تأكيد ربط حساب Business Manager بالمشروع
