### الظاهرة

عند تنفيذ `git push origin main` ظهر تحذير أحمر من GitHub:

```
remote: GitHub found 1 vulnerability on mesuef1974/qatar-university-advisor's
remote: default branch (1 moderate). To find out more, visit:
remote: https://github.com/mesuef1974/qatar-university-advisor/security/dependabot/1
```

رمز التعجب (!) ظهر على جميع عمليات الدفع الثلاث المنفذة في Sprint 08.

---

### التحليل التقني

**الحزمة المتأثرة:** `esbuild ≤ 0.24.2`
**معرف الثغرة:** GHSA-67mh-4wv8-2f99
**الرابط:** https://github.com/advisories/GHSA-67mh-4wv8-2f99

**طريقة الوصول للثغرة:**
```
vitest@2.x
  └── vite-node@2.x
        └── esbuild ≤ 0.24.2  ← الثغرة
  └── @vitest/mocker@2.x
        └── vite@2.x
              └── esbuild ≤ 0.24.2  ← الثغرة
@vitest/coverage-v8@2.x
  └── vitest@2.x  ← نفس السلسلة
```

**تأثير الثغرة:**
يمكن لأي موقع ويب زيارته المطور أثناء تشغيل `npm run dev` إرسال طلبات لـ Dev Server المحلي وقراءة استجاباته — بما قد يشمل ملفات المشروع.

**تقييم التأثير الفعلي:**
| البيئة | التأثير |
|--------|---------|
| Vercel Production | ✅ صفر — esbuild لا يعمل في الإنتاج |
| Dev Server المحلي | ⚠️ خطر عند التطوير المتزامن مع تصفح مواقع خارجية |
| CI/CD Pipeline | ✅ لا تأثير — يعمل في بيئة معزولة |

### خيارات الحل

#### الخيار أ — ترقية vitest إلى v3 ⭐ مُوصى به
```bash
npm install --save-dev vitest@^3 @vitest/coverage-v8@^3
npm audit  # يجب أن يعود بـ 0 vulnerabilities
```
**المزايا:** حل نهائي، esbuild محدَّث، دعم طويل الأمد
**المخاطر:** breaking change محتمل في API الاختبارات
**الإصدار الناتج:** v1.0.1 (patch)
**الوقت المقدر:** 30-60 دقيقة (شامل اختبار التوافق)

#### الخيار ب — Override esbuild مباشرة (مؤقت)
```json
// package.json
"overrides": {
  "esbuild": "^0.25.0"
}
```
**المزايا:** لا breaking changes في الاختبارات
**المخاطر:** قد يُحدث تعارضات، حل مؤقت غير مستدام
**الإصدار الناتج:** لا تغيير في الإصدار

#### الخيار ج — تجاهل مؤقت (غير مُوصى به)
```ini
# .npmrc
audit-level=high
```
**المزايا:** يُخفي التحذير فوراً
**المخاطر:** يُخفي المشكلة ولا يحلها — مخالف للمعايير المؤسسية
