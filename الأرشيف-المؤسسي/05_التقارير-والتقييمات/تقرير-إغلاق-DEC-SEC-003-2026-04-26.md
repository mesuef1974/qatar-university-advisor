## 2. الاكتشافات الحرجة أثناء التنفيذ

### 2.1 Migration 002 لم يُطبَّق إنتاجياً — اكتُشف اليوم

كانت Supabase تسجّل فقط `001_pgvector_semantic_search` — أي أن جدول `user_consents` لم يكن موجوداً → **PDPPL Article 7 معطَّل بالكامل قبل اليوم.**

### 2.2 Bug في Migration 003

كان migration 003 يحاول `DROP POLICY IF EXISTS "public_read"` لكن الأسماء الفعلية `universities_public_read` (إلخ). الإسقاط كان silent-fail → policies الجديدة `is_active=TRUE` مُضافة فوق القديمة `USING(TRUE)` = OR منطقي = الـ USING(TRUE) القديمة تظل تسرّب.

**الحل:** Migration 004 يستخدم الأسماء الصحيحة + يضمن إسقاط نظيف.

### 2.3 Vercel Auto-Deploys مُلغاة

- Deploy لـ commit `c01c03a` (DEC-SEC-003 docs) → CANCELED
- Deploy لـ commit `d59b644` (migration 004) → CANCELED

السبب غير مؤكَّد (ربما Vercel skip rules لـ docs-only أو git deployment lock). **لا تأثير على الإنتاج** — الإنتاج يعمل على آخر deploy ناجح (96e69fa) والـ DB hardened عبر MCP مباشرة.
